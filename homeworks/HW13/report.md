# Отчёт по HW13: Fine-tuning BERT для классификации эмоций

## 1. Кратко: что сделано
В рамках работы реализован полный пайплайн fine-tuning модели `bert-base-uncased` для задачи классификации текстов по шести эмоциям (sadness, joy, love, anger, fear, surprise). Выполнены следующие этапы:
- загрузка и анализ датасета `emotion`;
- демонстрация токенизации и работы готовой модели;
- дообучение BERT с ограниченным объёмом обучающих данных (2000 примеров);
- оценка качества на тестовой выборке, построение матрицы ошибок, сохранение предсказаний и артефактов.

## 2. Среда и воспроизводимость
- **Язык и библиотеки**: Python 3.10, PyTorch, Transformers, Datasets, scikit-learn, pandas, matplotlib, seaborn.
- **Фиксированный seed**: 42 (обеспечивает воспроизводимость).
- **Устройство**: CPU (в коде автоматически выбирается CUDA при наличии).
- **Модель**: `bert-base-uncased` (токенизатор и модель для классификации).
- **Гиперпараметры fine-tuning**:
  - эпохи: 1;
  - размер батча: train – 16, eval – 64;
  - максимальная длина последовательности: 32 токена;
  - стратегия оценки: после каждой эпохи;
  - критерий выбора лучшей модели: `f1_macro`;
  - early stopping: patience=2.

## 3. Данные и постановка задачи
- **Датасет**: `emotion` из библиотеки `datasets`.
- **Классы**: sadness, joy, love, anger, fear, surprise (6 классов).
- **Разбиение**: train – 16000, validation – 2000, test – 2000.
- **Особенность**: для ускорения экспериментов обучающая выборка была сокращена до 2000 примеров (первые 2000 записей).
- **Задача**: бинарная классификация (мультиклассовая) эмоции по тексту.

## 4. Токенизация и готовый инференс
- Токенизатор `bert-base-uncased` преобразует текст в `input_ids` и `attention_mask`. 
- Продемонстрирована токенизация нескольких примеров с выводом токенов, идентификаторов и специальных токенов `[CLS]`, `[SEP]`.
- Для инференса использована готовая модель `bhadresh-savani/bert-base-uncased-emotion`. На пяти тестовых фразах она правильно определила все эмоции (joy, sadness, love, anger, fear), что подтверждает пригодность BERT для данной задачи.

## 5. Fine-tuning и оценка
- **Предобработка**: токенизация с `padding="max_length"`, `truncation=True`, `max_length=32`. Удалён исходный текст, колонка `label` переименована в `labels`.
- **Модель**: `AutoModelForSequenceClassification` с `num_labels=6`.
- **Обучение**: использован `Trainer` из библиотеки Transformers с `TrainingArguments` (1 эпоха, логирование каждые 50 шагов, сохранение каждой эпохи).
- **Метрики**: accuracy и macro F1 вычисляются на валидации.
- **Результаты на тесте** (после fine-tuning):
  - Accuracy: 0.6095
  - Macro F1: 0.2943

Низкие метрики объясняются малым объёмом обучающей выборки (2000 вместо 16000) и всего одной эпохой обучения. При полном датасете можно ожидать значительного улучшения (как показывает готовая модель, достигающая >90% accuracy).

## 6. Результаты
- **Матрица ошибок** (сохранена в `artifacts/confusion_matrix.png`) показывает, что модель чаще всего путает `fear` с `sadness`, `love` с `joy`, а `surprise` распознаётся хуже всего.
- **Примеры предсказаний** (первые 100 тестовых образцов) сохранены в файле [`artifacts/sample_predictions.csv`](artifacts/sample_predictions.csv). Файл содержит: текст, истинную метку, предсказанную метку и уверенность (softmax-вероятность).
- **Анализ ошибок**: из 2000 тестовых примеров модель ошиблась в 781 случае (39.1%). Частые ошибки: `fear` → `sadness`, `anger` → `sadness`, `joy` → `sadness`. Это указывает на смещение модели в сторону класса `sadness` при недостатке данных.

## 7. Анализ
- **Сильные стороны**: пайплайн токенизации, обучения и оценки реализован корректно; артефакты сохранены.
- **Слабые стороны**: из-за сильного сокращения обучающей выборки и малого числа эпох модель не смогла выучить разделяющие признаки. Полученные метрики (0.61 accuracy) значительно ниже потенциально достижимых (>0.9).
- **Рекомендации**:
  - использовать полный тренировочный набор (16000 примеров);
  - увеличить число эпох (3–5);
  - применить более высокую максимальную длину последовательности (например, 64 или 128);
  - использовать более низкую скорость обучения (например, 2e-5) и планировщик.

## 8. Итоговый вывод
В ходе работы успешно реализован полный цикл fine-tuning BERT для классификации эмоций. Продемонстрирована токенизация, инференс готовой модели, дообучение на усечённом датасете и оценка качества. Несмотря на низкие абсолютные метрики (из-за ограниченных вычислительных ресурсов и намеренного сокращения данных), пайплайн является рабочим и может быть легко масштабирован на полный датасет для получения высоких результатов. Артефакты (матрица ошибок, примеры предсказаний) прилагаются.

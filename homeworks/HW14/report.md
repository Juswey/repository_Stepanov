# Отчёт по HW14: Эмбеддинги, FAISS, оценка retrieval и mini-RAG

## 1. Кратко: что сделано
Реализован учебный пайплайн векторного поиска по базе знаний (20 документов по NLP/ML). Выполнены:
- чанкинг текстов (50 слов, overlap 10);
- построение эмбеддингов моделью `all-MiniLM-L6-v2` с использованием GPU/CPU;
- создание индекса FAISS (IndexFlatIP);
- оценка retrieval на 10 контрольных запросах (hit@3, hit@5);
- эксперимент с параметром `top_k`;
- обновление базы знаний (добавлено 3 документа) и переиндексация;
- простой mini-RAG конвейер с шаблонной генерацией ответа;
- анализ ошибок retrieval.

## 2. Среда и воспроизводимость
- **Язык и библиотеки**: Python 3.10, numpy, pandas, matplotlib, faiss-cpu, sentence-transformers, torch, sklearn.
- **Фиксированный seed**: 42 (random, numpy, torch, CUDA).
- **Модель эмбеддингов**: `all-MiniLM-L6-v2` (sentence-transformers) с автоматическим выбором устройства (cuda/cpu).
- **Параметры чанкинга**: chunk_size = 50 слов, overlap = 10 слов.
- **Индекс FAISS**: `IndexFlatIP` на нормализованных векторах (эквивалент косинусного сходства).
- **Оценка retrieval**: 10 контрольных запросов, метрики hit@k (k=3,5).

## 3. База знаний и подготовка документов
- **База знаний**: 20 кратких статей по темам NLP, ML (BERT, GPT, attention, embedding, RAG и т.д.).
- **Чанкинг**: каждый документ разбит на чанки по 50 слов с перекрытием 10 слов. Получено 34 чанка.
- **Эмбеддинги**: модель `all-MiniLM-L6-v2` преобразует каждый чанк в 384-мерный нормализованный вектор.

## 4. Retrieval и индекс
- **Индекс FAISS**: `IndexFlatIP` (inner product) для нормализованных векторов, что эквивалентно косинусному сходству.
- **Функция поиска**: `search(query, k)` возвращает top-k чанков с оценками сходства.
- **Тестовые запросы**: например, "How does BERT work?" – первый релевантный результат (BERT) с score 0.648.

## 5. Эксперимент по параметрам retrieval и обновление базы знаний
- **Сравнение k=3 и k=5**: Hit@3 = 80%, Hit@5 = 90%. Для mini-RAG выбран k=3.
- **Обновление базы**: добавлены документы LoRA, Prompt Engineering, Vector Database. После переиндексации запросы "What is LoRA?" и др. стали находить новые документы.
- Результаты сохранены в `artifacts/retrieval_eval.csv` и `artifacts/retrieval_before_after_update.csv`.

## 6. Mini-RAG и результаты
- **Конвейер**: запрос → retrieval (top-3) → сборка контекста → генерация шаблонного ответа.
- **Примеры**:
  - Вопрос: "How can I fine-tune a large language model efficiently?" → ответ основан на чанках LoRA и Fine-tuning.
  - Вопрос: "What is the role of embeddings in search?" → ответ использует Semantic Search и Word Embeddings.
- Примеры сохранены в `artifacts/rag_examples.csv`.

## 7. Анализ
- **Успехи**: 8 из 10 контрольных запросов попали в top-3.
- **Ошибка**: запрос "What is zero-shot classification?" не получил hit@3 из-за слабого семантического совпадения. Возможные улучшения: увеличение overlap, реранкинг кросс-энкодером.
- **Влияние обновления**: добавление новых документов изменило выдачу для соответствующих запросов (например, "What is LoRA?" ранее не находил ничего, теперь находит LoRA).

## 8. Итоговый вывод
Все обязательные компоненты HW14 реализованы с учётом требований воспроизводимости (фиксация seed для random, numpy, torch; определение device). Артефакты сохранены в `artifacts/`. Работа демонстрирует полный пайплайн RAG: от чанкинга до генерации ответа на основе retrieval.

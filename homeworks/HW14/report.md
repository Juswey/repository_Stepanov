# Отчёт по HW14: Эмбеддинги, FAISS, оценка retrieval и mini-RAG

## 1. Кратко: что сделано
Реализован учебный пайплайн векторного поиска по базе знаний (20 документов по NLP/ML). Выполнены:
- чанкинг текстов (50 слов, overlap 10);
- построение эмбеддингов моделью `all-MiniLM-L6-v2`;
- создание индекса FAISS (IndexFlatIP);
- оценка retrieval на 10 контрольных запросах (hit@3, recall@3);
- эксперимент с параметром `top_k`;
- обновление базы знаний (добавлено 3 документа) и переиндексация;
- простой mini-RAG конвейер с шаблонной генерацией ответа;
- анализ ошибок retrieval.

## 2. Среда и воспроизводимость
- **Язык и библиотеки**: Python 3.10, numpy, pandas, matplotlib, faiss-cpu, sentence-transformers.
- **Фиксированный seed**: 42.
- **Модель эмбеддингов**: `all-MiniLM-L6-v2` (sentence-transformers).
- **Параметры чанкинга**: chunk_size = 50 слов, overlap = 10 слов.
- **Индекс FAISS**: `IndexFlatIP` на нормализованных векторах (эквивалент косинусного сходства).
- **Оценка retrieval**: 10 контрольных запросов, метрики hit@k и recall@k (k=3,5).

## 3. Данные и постановка задачи
- **База знаний**: 20 кратких статей по темам NLP, ML (BERT, GPT, attention, embedding, RAG и т.д.).
- **Объём**: 34 чанка после разбиения.
- **Задача**: retrieval релевантных фрагментов по текстовому запросу и генерация ответа на основе найденного контекста (mini-RAG).

## 4. Чанкинг и эмбеддинги
- Чанкинг фиксированного размера в словах с перекрытием обеспечивает сохранение контекста между фрагментами.
- Модель `all-MiniLM-L6-v2` даёт 384-мерные нормализованные эмбеддинги, подходящие для семантического поиска.

## 5. Retrieval и оценка
- **Контрольные запросы**: 10 вопросов с явно указанными релевантными документами (по ID).
- **Метрики**:
  - Hit@3 = 0.80 (8 из 10 запросов получили релевантный документ в топ-3)
  - Hit@5 = 0.90 (9 из 10)
- Результаты сохранены в [`artifacts/retrieval_eval.csv`](artifacts/retrieval_eval.csv).

## 6. Эксперимент с параметрами
Сравнение `k=3` и `k=5` показало ожидаемый рост hit rate с 80% до 90%. Для mini-RAG выбран `k=3` как компромисс между полнотой и объёмом контекста.

## 7. Обновление базы знаний
- Добавлены документы по темам LoRA, Prompt Engineering, Vector Database.
- После переиндексации retrieval по новым запросам стал релевантным (см. [`artifacts/retrieval_before_after_update.csv`](artifacts/retrieval_before_after_update.csv)).
- Сравнение до/после показывает изменение источников для запросов, связанных с новыми документами.

## 8. Mini-RAG
- Конвейер: запрос → retrieval (top-3) → сборка контекста → генерация ответа.
- Генерация шаблонная (вывод контекста), но структура соответствует учебному RAG.
- Примеры работы сохранены в [`artifacts/rag_examples.csv`](artifacts/rag_examples.csv).

## 9. Анализ ошибок
- Запрос "What is zero-shot classification?" не получил hit@3 из-за слабого семантического совпадения с единственным чанком по этой теме. Возможные улучшения: увеличение overlap, более точная формулировка запроса, использование кросс-энкодера для реранкинга.
- Остальные запросы обрабатываются корректно.

## 10. Итоговый вывод
Все обязательные компоненты HW14 реализованы: чанкинг, эмбеддинги, FAISS, оценка retrieval, эксперимент, обновление базы и mini-RAG. Артефакты сохранены в папке `artifacts/`. Работа демонстрирует понимание базового retrieval-augmented generation и метрик качества поиска.

# Отчёт по HW14: Эмбеддинги, FAISS, оценка retrieval и mini-RAG

## 1. Кратко: что сделано
Реализован учебный пайплайн векторного поиска по базе знаний (20 документов по NLP/ML). Выполнены:
- чанкинг текстов (50 слов, overlap 10);
- построение эмбеддингов моделью `all-MiniLM-L6-v2` с использованием GPU/CPU;
- создание индекса FAISS (IndexFlatIP);
- оценка retrieval на 10 контрольных запросах (**hit@3, hit@5, recall@3, recall@5**);
- эксперимент с параметром `top_k`;
- обновление базы знаний (добавлено 3 документа) и переиндексация;
- простой mini-RAG конвейер с шаблонной генерацией ответа и отображением источников (source tracking);
- анализ ошибок retrieval.

## 2. Среда и воспроизводимость
- **Язык и библиотеки**: Python 3.10, numpy, pandas, matplotlib, faiss-cpu, sentence-transformers, torch, sklearn.
- **Фиксированный seed**: 42 (random, numpy, torch, CUDA).
- **Модель эмбеддингов**: `all-MiniLM-L6-v2` (sentence-transformers) с автоматическим выбором устройства (cuda/cpu).
- **Параметры чанкинга**: chunk_size = 50 слов, overlap = 10 слов.
- **Индекс FAISS**: `IndexFlatIP` на нормализованных векторах (эквивалент косинусного сходства).
- **Оценка retrieval**: 10 контрольных запросов, метрики hit@k и recall@k (k=3,5).

## 3. База знаний и подготовка документов
- **База знаний**: 20 кратких статей по темам NLP, ML (BERT, GPT, attention, embedding, RAG и т.д.).
- **Чанкинг**: каждый документ разбит на чанки по 50 слов с перекрытием 10 слов. Получено 34 чанка.
- **Эмбеддинги**: модель `all-MiniLM-L6-v2` преобразует каждый чанк в 384-мерный нормализованный вектор.
- **Метаданные**: Для каждого чанка сохранен идентификатор исходного документа (doc_id) и позиция чанка для обеспечения трекинга источников.

## 4. Retrieval и индекс
- **Индекс FAISS**: `IndexFlatIP` (inner product) для нормализованных векторов, что эквивалентно косинусному сходству.
- **Функция поиска**: `search(query, k)` возвращает top-k чанков с оценками сходства **и соответствующими метаданными источников**.
- **Тестовые запросы**: например, "How does BERT work?" – первый релевантный результат (BERT) с score 0.648.

## 5. Эксперимент по параметрам retrieval и обновление базы знаний
- **Сравнение k=3 и k=5**: 
    - Hit@3 = 80%, **Recall@3 = 80%**
    - Hit@5 = 90%, **Recall@5 = 90%**
    - *Примечание: В условиях одного релевантного документа на запрос значения Hit и Recall совпадают.*
    - Для mini-RAG выбран k=3.
- **Обновление базы**: добавлены документы LoRA, Prompt Engineering, Vector Database. После переиндексации запросы "What is LoRA?" и др. стали находить новые документы.
- Результаты сохранены в `artifacts/retrieval_eval.csv` и `artifacts/retrieval_before_after_update.csv`.

## 6. Mini-RAG, источники и результаты
- **Конвейер**: запрос → retrieval (top-3) → сборка контекста → генерация шаблонного ответа → форматирование вывода с перечислением источников.
- **Трекинг источников (Source Tracking)**: 
    - При формировании ответа извлекаются `doc_id` найденных чанков.
    - Пользовательский вывод содержит не только сгенерированный текст, но и блок **"Источники:"**, где перечислены названия документов/чанков, на основе которых построен ответ.
- **Примеры**:
  - Вопрос: "How can I fine-tune a large language model efficiently?" → ответ основан на чанках LoRA и Fine-tuning. **Источники: [doc_21_LoRA], [doc_4_Fine-tuning].**
  - Вопрос: "What is the role of embeddings in search?" → ответ использует Semantic Search и Word Embeddings. **Источники: [doc_12_Word_Embeddings], [doc_15_Semantic_Search].**
- Примеры сохранены в `artifacts/rag_examples.csv` (включая колонку с перечнем использованных doc_id).

## 7. Анализ
- **Успехи**: 8 из 10 контрольных запросов попали в top-3. Значения Recall@3 и Recall@5 демонстрируют способность системы находить релевантную информацию в большинстве случаев.
- **Ошибка**: запрос "What is zero-shot classification?" не получил hit@3 и, следовательно, recall@3=0 из-за слабого семантического совпадения. Возможные улучшения: увеличение overlap, реранкинг кросс-энкодером.
- **Влияние обновления**: добавление новых документов изменило выдачу для соответствующих запросов (например, "What is LoRA?" ранее не находил ничего, теперь находит LoRA и корректно указывает источник).

## 8. Итоговый вывод
- Все обязательные компоненты HW14 реализованы с учётом требований воспроизводимости (фиксация seed для random, numpy, torch; определение device). Добавлена оценка полноты (recall@k) и механизм указания источников информации в ответах RAG. Артефакты сохранены в `artifacts/`. Работа демонстрирует полный пайплайн RAG: от чанкинга до генерации ответа на основе retrieval с подтверждением источников.
